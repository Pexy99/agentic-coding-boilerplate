# Contributing Guide (Human)

이 가이드는 사람이 이 프로젝트를 운영하고 에이전트와 협업하는 방법을 설명합니다.

## 1. 작업 시작 (New Track)
모든 새로운 기능이나 버그 수정은 새로운 **트랙(Track)**으로 시작합니다.
1. `docs/tracks.md`에 새로운 트랙을 등록합니다.
2. `docs/tracks/<track_id>/` 디렉터리를 생성합니다.
3. `spec.md`와 `plan.md`를 작성하여 구현할 내용과 단계를 정의합니다.

## 2. 문서 관리 원칙
- **선 문서, 후 코드**: 코드를 수정하기 전 반드시 관련 문서(`product.md`, `tech-stack.md`, `spec.md` 등)를 먼저 업데이트해야 합니다.
- **중복 금지**: 동일한 정보가 여러 곳에 기록되지 않도록 하십시오.
- **ADR 작성**: 중요한 구조적 결정은 `docs/adr/` 아래에 기록합니다.

## 3. 에이전트 활용
- 에이전트에게 작업을 맡길 때는 `AGENTS.md`를 읽게 하여 워크플로우를 강제하십시오.
- 에이전트가 작성한 코드는 반드시 사람이 리뷰하고 최종 책임을 집니다.

## 4. 브랜치 및 PR 전략
- `main` 브랜치는 항상 배포 가능한 상태를 유지합니다.
- 모든 작업은 트랙 ID를 포함한 브랜치에서 진행하며 PR을 통해 병합합니다.
- 병합 전 모든 테스트와 린트(`docs/workflow.md` 참조)가 통과되어야 합니다.
